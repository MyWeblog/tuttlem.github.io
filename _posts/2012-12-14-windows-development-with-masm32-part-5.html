Introduction

Welcome to the fifth instalment in the series on writing windows applications using MASM32. We've covered the basics pretty thoroughly in parts 1, 2 and 3. We took a bit of a left turn for part 4 covering double buffering which admittedly is a bit of an advanced topic.

We're back on the straight and narrow now - today we're going to discuss common controls.
 

What are they?

Window common controls are served out of a dll called Comctl32.dll. The fill list can be found here and there's heaps in there: toolbar, tooltip, status bar, property sheet, treeview, trackbar, the list goes on.
 

Show me!

Today, we're just going to create a simple, humble, common status bar for our window. First up, we need to import another header and library. Nicely enough these are named comctl32.inc and comctl32.lib respectively.


Next up we have a touch of initialization to get the library in gear. This is just a one-liner.


So, InitCommonControls just initializes the window classes involved in using common controls. Now we'll use the WM_CREATE message in the window procedure to actually create our child controls. In our case today, it's just going to be the status window. So, we'll use CreateStatusWindow.



We can fire our application up now and see our statusbar in all its glory at the bottom of the window. This isn't really enough though. I really need to show you how to put something in the status window. A good case here, I think is just some text. I'm sure that you will find much more useful things to do with this, but for this purposes of this post, when a user clicks in the window area with their left mouse button, I'm going to put the string "Click!" into the status bar.

Note the use of SendMessage. This is how we'll instruct all of our UI objects to do things of interest for us. Sending messages is how we all co-operate in windows land.


So, that's it for a very tip-of-the-ice-berg (and it's a massive one) look at common controls.
